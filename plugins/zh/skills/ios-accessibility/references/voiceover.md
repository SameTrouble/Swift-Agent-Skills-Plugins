# VoiceOver

iOS VoiceOver 无障碍的核心概念。

## 什么是 VoiceOver

VoiceOver 是一种屏幕阅读器，它朗读界面元素并启用手势导航，主要（但不仅限于）面向失明或低视力用户。

## 焦点和激活基础

- **单击** 将焦点移动到被点击的元素并朗读它。
- **双击** 激活聚焦的元素（按钮、开关、链接）。

这就是为什么标签、值和特质很重要：VoiceOver 依赖它们来描述元素是什么以及如何与之交互。

## 探索 vs 滑动导航

VoiceOver 用户主要通过两种方式导航：

- **触摸探索** — 在屏幕上拖动手指以听到手指下方的内容。
- **滑动导航** — 左右滑动以按结构化顺序移动浏览元素（上一个/下一个）。

当你将元素分组良好并设置合理的顺序时，滑动导航变得快速且可预测。

## 核心无障碍属性

可访问元素的核心属性（标签是唯一应强制要求的，除非有非常好的理由不这样做——在这种情况下它至少应有值）：

| 属性 | 用途 | 示例 |
|----------|---------|---------|
| **Label** | 元素名称 | "Rating" |
| **Value** | 当前状态 | "5 stars" |
| **Hint** | 操作结果或额外上下文 | "Rates how much you like the conference from 1 to 5 stars" |
| **Traits** | 角色和状态 | adjustable |

### 标签

- 简洁且具描述性（考虑上下文）
- 不包含控件类型（"Play" 而非 "Play button"）
- 为所有支持的语言本地化
- 尽可能匹配可见文本（同时帮助 Voice Control）

### 值

用于有变化状态的控件：
- 滑块："50 percent"
- 开关："On" / "Off"
- 步进器："3 of 5"

### 提示

谨慎使用 — 仅当操作在标签和特质之上需要额外上下文时。
尽可能将提示表述为结果而非命令，例如："Deletes this item from your list."

### 特质

传达角色和状态：
- **Button** — 双击激活
- **Header** — 章节标题（启用转子导航）
- **Selected** — 当前选择的项目
- **Adjustable** — 上下滑动更改值
- **Link** — 打开 URL
- **Image** — 图片内容
- **Disabled (not enabled)** — 不可用控件

使用实现文件中框架特定的 API 名称：
- SwiftUI：`references/voiceover-swiftui.md`
- UIKit：`references/voiceover-uikit.md`

## 分组

将相关元素分组以：
- 减少滑动次数
- 提供更好的上下文
- 简化导航

模式：
- **自动合并（SwiftUI `.combine`）** — 子元素合并为一个朗读元素
- **手动合并** — 将组视为一个元素并自己提供标签/值/特质
- **语义分组 / 分组遍历** — 保持子元素单独可访问同时保留组上下文

这些模式的 SwiftUI API 名称是 `.combine`、`.ignore` 和 `.contain`。UIKit 使用 `isAccessibilityElement` 进行手动合并，使用 `shouldGroupAccessibilityChildren` 进行分组遍历（子元素保持独立，但 VoiceOver 在移动之前会遍历该组）。当需要自定义读取顺序时使用显式 `accessibilityElements` 排序。

## 导航顺序

VoiceOver 默认从左到右、从上到下导航。仅当视觉布局与逻辑任务流程不匹配时才覆盖。

## 自定义操作

暴露次要操作（删除、分享等）而不使界面杂乱。操作出现在 VoiceOver 的操作转子中。理想情况下，这些操作也应独立可访问，例如在详情屏幕或溢出菜单中。

## 补充内容

对于数据丰富的 UI，提供额外上下文作为自定义无障碍内容，而非重载主标签/值。首先将最重要的状态放在标签/值/特质中，然后将次要细节作为补充内容添加。尽可能使相同的信息在其他地方独立可访问（例如在详情屏幕中）。

## 焦点和公告

**移动焦点在以下情况后：**
- **屏幕切换** — 仅当新屏幕是显著覆盖当前屏幕或阻止下方屏幕交互的自定义覆盖层时（例如自定义模态弹出）
- **新内容出现** — 在采取操作后（布局更改）

**公告：**
- 用户可能错过的重要事件（错误状态、长时间运行的任务完成）

## 手势

| 手势 | 操作 |
|---------|--------|
| 轻点 | 选择元素 |
| 双击 | 激活 |
| 右/左滑动 | 导航下一个/上一个元素 |
| 上/下滑动 | 调节或按转子导航 |
| 双指旋转 | 更改转子模式 |
| 双指双击 | Magic Tap（主要操作） |
| 双指擦洗 | Escape（关闭） |
| 三指滑动 | 滚动 |
| 三指双击 | 静音/取消静音 VoiceOver 语音 |
| 三指三次点击 | 屏幕变暗 |

## 测试

**启用 VoiceOver：**
- **无障碍快捷方式**（如已配置则最快）：
  - 三次点击侧边按钮（或旧设备上的主屏幕按钮）
  - 如果无障碍快捷方式中启用了多个功能，从菜单中选择 VoiceOver
  - 配置位置：设置 > 无障碍 > 无障碍快捷方式 > VoiceOver
- **设置**：设置 > 无障碍 > VoiceOver > 切换开启
- **Siri**："Hey Siri, turn on VoiceOver"（或 "turn off VoiceOver"）

**测试步骤：**
1. 浏览每个屏幕
2. 验证标签、值、特质和顺序
3. 所有内容被传达且每个操作都可执行
4. 通过操作转子测试自定义操作
5. 如果你有信心，用屏幕变暗测试（三指三次点击）以体验没有视觉提示的情况
