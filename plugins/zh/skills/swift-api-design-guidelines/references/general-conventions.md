# 通用约定

## 文档化计算属性复杂度
- 如果计算属性不是 `O(1)`，文档化其复杂度。
- 许多读者会假设属性访问是廉价的，除非另有说明。

## 优先使用方法/属性而非自由函数
仅在以下情况使用自由函数：
1. 没有明显的 `self`。
2. 函数是无约束泛型。
3. 函数语法是既定的领域记法。

## 遵循 Swift 大小写约定
- 类型/协议：`UpperCamelCase`。
- 其他声明：`lowerCamelCase`。
- 首字母缩略词和首字母简写应与风格约定保持一致的大小写。

```swift
var utf8Bytes: [UTF8.CodeUnit]
var isRepresentableAsASCII = true
var radarDetector: RadarScanner
```

## 重载
- 当含义相同或领域不同时，方法可共享基名。
- 不要为语义不同的操作复用基名。
- 避免仅靠返回类型重载；类型推断会使调用变得含糊。

## 审查启发式
- 检查重载集合在调用处是否可读、语义连贯且无歧义。
