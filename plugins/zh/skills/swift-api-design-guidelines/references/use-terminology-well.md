# 善用术语

## 优先清晰而非晦涩
- 当常见词汇能保留含义时，避免使用晦涩术语。
- 仅在专业术语能提供必要精确性时使用它们。

## 保留既定含义
- 如果使用专业术语，保留其公认含义。
- 不要以令专家惊讶或误导新手的方式重新定义术语。

## 避免不必要的缩写
- 非标准缩写会变成隐性的专业术语。
- 仅在领域实践中广泛确立时使用缩写。

## 采纳惯例
- 遵循 Swift 及更广领域中的既有技术词汇。
- 当文化标准名称能增进理解互操作性时，选择它们。

示例：
- 优先使用 `Array`，而非发明新的简化同义词。
- 在数学密集的 API 中，`sin(x)` 比过度解释名称更好。
