# CLAUDE.md

本文件为 Claude Code（claude.ai/code）在此仓库中工作时提供指导。

## 这个仓库是什么

一个单一的 Claude Code **技能**（`ios-code-audit`），为 iOS/macOS Swift 项目生成 `CODE_AUDIT.md` 报告。这里没有应用代码、构建系统或测试套件——"交付物"是驱动对*其他* Swift 代码库进行审计运行的提示词。

当用户调用本技能时，Claude 运行在用户的 iOS 项目目录中，*而不是*这个仓库。在此仓库中的编辑只会改变未来审计的行为。

## 文件布局

- `SKILL.md` —— 技能定义，以及 Claude 被调用时遵循的 6 步工作流程。这是事实来源；其他一切都从它引用。
- `references/agent-prompts.md` —— 三个并行 Explore 代理（并发/API 现代化、死代码/重复代码、缺陷/安全/性能）的可复制粘贴简报。底部有一个占位符表，说明每个 `{PLACEHOLDER}` 是如何发现的。
- `references/report-template.md` —— 带章节编号的 `CODE_AUDIT.md` 骨架。章节编号（3.1、3.2、……）在设计上跨编辑保持稳定，因此用户引用"§5.4"的笔记不会错位。

这三个文件构成一条流水线：`SKILL.md` 驱动工作流程，代理简报产出原始发现项，报告模板对它们进行结构化。

## 本技能特有的编辑规则

- **严重性纪律是承重不变量。** `SKILL.md` 强制执行四级指南（Critical/High/Medium/Low），并*要求*在每条 Critical 主张写入报告之前进行抽查核实。不要放松这一点——这是对以往运行中代理夸大严重性的直接回应。
- **三个代理必须并行运行。** `agent-prompts.md` 反复强调："在单条消息中通过多次 `Agent` 工具调用发送全部三个代理。"如果你编辑工作流程，请保留并行启动的要求。
- **逐条发现项的模板在三个代理简报中完全相同。** 如果你更改了格式（位置/问题/原因/行动/严重性），请在三个简报*以及*报告模板中同步更新——综合过程依赖于它保持机械化。
- **`Dead/`（或任何显式归档的目录）默认被排除。** 本技能期望读取被审计项目自身的 `CLAUDE.md` 来发现被排除的路径。

## 技能显式不涵盖的内容

列于 `SKILL.md` 的"本技能不涵盖的内容"下——Metal 内核正确性、Xcode 项目结构、第三方依赖、深度测试覆盖、本地化措辞、Instruments 剖析。如果范围发生变化，请保持该列表同步。
