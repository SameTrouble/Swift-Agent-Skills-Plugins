# 预览

## 意图

用预览验证布局、状态连接和注入依赖，而无需依赖运行中的应用或实时服务。

## 核心规则

- 为主要状态加上 `#Preview` 覆盖，以及重要的次要状态如加载、空、错误。
- 使用确定性夹具、mock 和样本数据。不要让预览依赖实时网络调用、真实数据库或全局单例。
- 在预览中直接安装所需环境依赖，使视图能隔离渲染。
- 保持预览设置靠近视图，直到变得嘈杂；然后提取轻量预览辅助或夹具。
- 如果预览崩溃，先修复状态初始化或依赖连接，再扩展功能。

## 示例：简单预览状态

```swift
#Preview("Loaded") {
  ProfileView(profile: .fixture)
}

#Preview("Empty") {
  ProfileView(profile: nil)
}
```

## 示例：带注入依赖的预览

```swift
#Preview("Search results") {
  SearchView()
    .environment(SearchClient.preview(results: [.fixture, .fixture2]))
    .environment(Theme.preview)
}
```

## 预览清单

- 预览是否安装了每个所需的环境依赖？
- 是否覆盖了至少一个成功路径和一个非正常路径？
- 夹具是否稳定且小到能快速阅读？
- 预览能否在没有网络、认证或应用全局初始化的情况下渲染？

## 陷阱

- 不要通过让依赖可选来掩盖预览崩溃，如果生产视图需要它们。
- 当命名样本更易读时，避免庞大内联夹具。
- 除非项目别无选择，否则不要把预览耦合到全局共享单例。
