# SwiftUI 无障碍清单

本清单源自 Apple 官方无障碍指南，用于在应用 SwiftUI 无障碍审核器建议的改动后进行**手动验证**。

在适用场景下使用 VoiceOver、Dynamic Type 和键盘导航进行验证。

---

## VoiceOver 与语义
- [ ] 所有仅图标按钮都有清晰、有意义的标签
- [ ] 标签尽可能匹配可见文字，以确保 Voice Control 命令可预测
- [ ] 无重复播报（父元素与子元素播报相同文字）
- [ ] 标题正确暴露为标题
- [ ] 相关元素在适当时被逻辑分组
- [ ] 阅读顺序与视觉和逻辑布局一致
- [ ] 自定义可点击视图同时暴露按钮语义和激活操作

## Dynamic Type
- [ ] 文字可正确缩放至最大无障碍尺寸
- [ ] 重要信息未因截断而丢失
- [ ] 布局自然适应，不依赖 minimumScaleFactor
- [ ] 无固定字号阻碍文字缩放

## 焦点与键盘导航（macOS / iPad）
- [ ] 界面可完全仅用键盘操作
- [ ] 焦点顺序可预测且符合逻辑
- [ ] 自定义组件可在需要时接收焦点
- [ ] 焦点在交互后不会被困住或丢失
- [ ] 自定义操作可在不使用仅触控手势的情况下触达

## 颜色与对比度
- [ ] 信息不仅依靠颜色传达
- [ ] 状态（错误、选中、禁用）在没有颜色的情况下也可理解
- [ ] 尽可能优先使用系统或语义颜色

## 触控目标（iOS）
- [ ] 可点击元素至少约 44x44 pt
- [ ] 需要时在不改变视觉布局的前提下扩展点击区域
- [ ] 自定义可点击容器仍可通过 VoiceOver 激活

## Voice Control 与 Switch Control（iOS / iPadOS）
- [ ] Voice Control "Show names" 暴露清晰、不重复的标签
- [ ] Switch Control 能以逻辑扫描顺序触达所有交互元素
- [ ] 分组减少不必要的扫描停留，同时不隐藏操作

## 动态效果
- [ ] 动画微妙且不阻碍交互
- [ ] 在适用场景下尊重 Reduce Motion 偏好

## WWDC26 / SDK 2027 准备
- [ ] 调整大小、iPhone Mirroring、iPad windowing 和工具栏溢出时文字保持可读且焦点顺序稳定
- [ ] Liquid Glass、半透明材质和 scroll edge effects 在 Reduce Transparency 和 Increase Contrast 下保持清晰可读
- [ ] 重排、拖放、滑动操作和手势优先的流程暴露无障碍替代方案
- [ ] 媒体界面暴露字幕选择并尊重系统字幕样式
- [ ] 功能、标签页、菜单和操作名称清晰、可本地化，且可用 Voice Control 操作

---

## 最终验证
- [ ] 启用 VoiceOver 后界面可用
- [ ] 在极端文字尺寸下界面仍可用
- [ ] 未引入新的无障碍回归
