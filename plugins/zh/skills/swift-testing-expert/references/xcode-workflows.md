# Xcode 工作流

## 何时使用此参考

在 Xcode 中快速调试失败、配置聚焦的测试计划，以及从大型测试报告中提取洞察时使用此文件。

## 测试导航器使用

- 在函数、套件、Tag 和参数级运行测试。
- 在参数化测试中，仅重跑失败的参数以快速迭代。
- 使用"按 Tag 分组"快速检查跨套件行为。

### 示例流程

1. 运行套件。
2. 打开失败的参数化参数。
3. 仅重跑该参数以快速迭代。

## 过滤与分组

- 在导航器中使用 Tag 过滤器进行聚焦的开发循环。
- 保持 Tag 命名稳定，便于团队复用过滤器和计划。
- 优先使用基于 Tag 的包含/排除而非脆弱的测试名模式。

### 建议的 Tag 约定

- `core` - 始终开启的快速检查
- `integration` - 外部依赖覆盖
- `regression` - Bug 修复锁定测试
- `flaky` - 修复期间的临时隔离

## 测试计划

- 在测试计划中按 target 配置包含/排除 Tag。
- 组合过滤器时有意区分"任意 Tag"与"所有 Tag"。
- 为以下场景维护独立计划：
  - 快速核心检查
  - 集成检查
  - 较慢/可选场景

### 示例计划策略

- `Core` 计划：包含 `core`，排除 `integration`。
- `Integration` 计划：包含 `integration`，排除 `flaky`。
- `ReleaseGate` 计划：包含 `core` 和 `regression`。

## 报告分诊

- 查看按 Tag/Bug/目标的分布洞察以发现失败聚集。
- 优先调查分组失败（通常表明系统性回归）。
- 确保禁用/已知问题的原因在报告中可见且可操作。

### 分诊步骤

1. 检查失败是否按共享 Tag 聚集。
2. 打开一个代表性失败。
3. 确认根因是通用的（依赖/故障/配置）还是测试局部的。
4. 修复根因，然后移除临时的已知问题标注。

## 诊断质量

- 保持期望富有表现力且范围狭窄。
- 改善参数/类型描述以加快根因识别。
- 确保 Bug Trait 关联到可跟踪的问题。

## 清单

- Tag 命名在跨套件间一致。
- 测试计划反映团队工作流（本地开发、CI、发布）。
- 参数化失败在大范围重跑前先在参数级重跑。
